# 01 — 介绍:这是给谁用的工具

> **审计:** 2026-06-04 · 章节与 v0.4.0 同步 · 全部示例通过 · 全部链接有效
## 这是什么

`inp_tool` 是一个用来**批量生成 CFD++ `mcfd.inp` 算例**的命令行工具。给它一个样例 `.inp` 文件,它会:

1. 扫描一组参数(比如 6 个攻角 × 2 个马赫数)
2. 自动生成 12 个变体,每个里面攻角/马赫都被正确改写
3. 同时算出对应的速度分量(气动学里叫"几何分解")
4. 输出一份 `manifest.json` 索引,记录所有 case 的关键参数

## 解决什么问题

飞行器气动参数研究、进气道畸变、翼型结冰……这类研究通常要跑几十甚至几百个算例,每个算例改几个数字。以前:

- 手工复制 + 文本编辑器改,容易改错
- 不记得之前改了什么(没记录)
- 想批量改又不会写脚本

`inp_tool` 把这个流程固化成命令,几秒钟出几百个算例,带审计索引。

## 谁该用

- 飞行器总体 / 气动 / 进气道工程师
- 做参数扫描、做 DOE 的人
- 任何要批量生成 `.inp` 算例的人

**前提:** 你的 `.inp` 样例里至少有 `guiopts` 和/或 `physics` 块,`inp_tool` 才能正确改字段。

## 一次典型的工作流

```
你提供:  1 个 mcfd.inp 样例
                  +
         1 份 JSON/YAML 扫描配置(告诉它扫哪些参数)
                  ↓
inp-tool 解析 → 笛卡尔积展开 → 改写每个 case
                  ↓
你得到:  N 个 mcfd_v1.inp, mcfd_v2.inp, ... + manifest.json
                  ↓
         把这些 .inp 喂给 CFD++ 求解器
```

## 和 CFD++ 求解器的关系

`inp_tool` **不调用求解器**。它只生成 `.inp` 文件。生成完之后:

- Linux: 用 `mcrun` 或自己写的 shell 脚本批量跑
- Windows: CFD++ GUI 自带的"批量提交"功能
- HPC: 自己的调度器(SLURM / PBS / LSF)

`inp_tool` 关心的是"怎么快速生成对的 `.inp`",不关心"怎么跑 CFD"。

## 看到这里你该知道

- ✅ 工具名:`inp-tool`(命令) / `inp_tool`(Python 包)
- ✅ 安装位置:Python 虚拟环境(项目推荐 `conda create -n cfdchanger python=3.8`)
- ✅ 输出:多个 `.inp` + 一份 `manifest.json`
- ❌ 不会自动跑 CFD++ 求解
- ❌ 不会画图(那是后处理的事)
